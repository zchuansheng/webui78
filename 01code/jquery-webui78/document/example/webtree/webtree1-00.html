webTree插件的使用说明

名称：webTree

一、定义与说明

webtree是基于ul,li标签实现的树控件，实现了树结构视图的功能。它可以把一段树形结构的html代码格式化树，也可以动态生成树结构

（一）从静态html代码格式化成树视图 ，您需要完成以下工作：

1、编写ul,li标签的html代码(适用于静态加载)

2、在页面加载完成时，调用JS，完成对tree的效果渲染，及参数设置

（二）动态加载树节点：

1、确定树的根目录位置

2、在页面加载完成时，调用JS，完成对tree的效果渲染，及参数设置，要设置加载的URL等

3、调用loadNode加载第一级节点

详细请参见示例页面

id:是节点的唯一标识，不应有重复

name:用于显示的节点文本名称

isleaf:是否是叶子节点,如果true或字符"1"，则节点前无展开符号

parent_id：当前节点的父节点

param：加载当前节点的子节点时，需提交后台的附加参数对象（名:值对象）

check_status:设置节点的初始选中状态:"1":选中；,"0"：未选中，check_status仅适用于带checkbox或radio的树

form_used boolean false （缺省） 动态加载节点时，是否同步提交表单数据 $('#tree").webtree({form_used:true});
lockedOnLoad boolean false （缺省） 当动态加载节点时，是否遮罩整个树

true：加载数据时，整个树不能被点击，

false:加载数据时，仅当前节点不能被点击
onPreLoad 函数 null(缺省) 设置节点加载前的回调函数

格式为:function onPreLoad(jsdata,ops)

jsdata：后台的json对象

ops:树的配置信息

格式为functiononLoadFinished(node,jsdata)

node:加载的节点（即加载前点击的节点）

jsdata：后台加载的json数据见节点的json对象

注意： if(node.attr("id")=="dept_tree"){ //此语句可用于判断当前节点是否根节点，"dept_tree"为自定义的树ID

}
onCollapse 函数 null(缺省) 设置节点折叠后的回调函数

格式为function onCollapse(node)

node:被折叠的节点
onExpand 函数 null(缺省) 设置节点展开后的回调函数

格式为function onExpand(node)

node:被展开的节点
onSelect 函数 null(缺省) 设置节点被选中后的回调函数

格式为function onSelect(node)

node:被选中的节点
onCheckClick 函数 null(缺省) 设置节点的check box被点击时的回调函数

格式为function onCheckClick(node,newStatus)

node:被选中的节点

newStatus:节点的当前状态：1为选中，0为未选中
